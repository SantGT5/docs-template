# Quick Start

Adding a new documentation page takes two steps: create the content, then
register it in the navigation.

## Add a page

<Steps>

<Step title="Create an MDX file">

Add a file under `src/pages/docs/content/`, for example `guides.mdx`:

```mdx
# Guides

Write **Markdown** here, and use components when you need them.

<Callout type="info">
  Standard Markdown is automatically styled to match the theme.
</Callout>
```

</Step>

<Step title="Register it in the navigation">

Open `src/config/navigation.ts` and add an entry. This drives the sidebar link,
the route, and prev/next paging — all at once.

```ts
{
  title: "Guides",
  path: "/docs/guides",
  load: () => import("@/pages/docs/content/guides.mdx"),
}
```

</Step>

</Steps>

<Callout type="tip">

The page at the path `/docs` is treated as the documentation index. Every other
path becomes a nested route under it.

</Callout>

## Using components in MDX

Custom components are registered globally, so you can use them in any `.mdx`
file **without importing them**:

```mdx
<CommandBox>npm run build</CommandBox>

<Callout type="warning" title="Careful">
  This action cannot be undone.
</Callout>
```

See the [Components](/docs/components) page for the full list and live examples.

## Theming

All accents are driven by a single `brand` color ramp in `src/theme/index.ts`.
Change the ramp to re-brand the entire site:

```ts
colors: {
  brand: {
    500: { value: "#0ea5e9" },
    600: { value: "#0284c7" },
    // …
  },
},
```
